git使用

发表于2023-08-17|更新于2023-08-18|学习笔记工具使用知识快查
|字数总计:493|阅读时长:1分钟|阅读量:

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

概念

git工作流程

工作区 暂存区 版本库

工作区

一个仓库中除.git外的所有文件

暂存区

stage或index，称为索引，一般存放在.git/index

版本库

.git即为版本库

objects可以理解为储存分支信息与索引信息的数据库

git命令

git checkout -b newbranch创建新分支并切换

git merge branch1 将branch1合并到当前分支

git rebase branch1 类似于将当前分支合并到branch1

^上一个提交记录~num上n个提交记录

git branch -f branch commitHistory将branch强制移动到commitHistory提交上

git reset branch撤销更改到branch

git revert branch撤销更改到branch

区别：

reset不可用于远程仓库

reset相当于修改历史，revert原理是进行一个新的提交，修改至想要修改的提交

git status查看暂存区状态

git add file将file添加到暂存区

git rm file将file从暂存区和工作区删除

git branch查看分支

git branch newBranch创建一个新的分支newBranch

git checkout -- file使用暂存区的file覆盖当前工作区中的file

git checkout .该命令与上一个类似，只不过该命令会使用所有暂存区的文件进行覆盖

该操作很危险，会清除当前未提交的改动

git checkout branchName切换到分支branchName

该操作很危险，会清除当前未提交的改动

git reset回退版本

git reset 命令 | 菜鸟教程

git switch分支操作

cmd means
git switch branch1 切换到branch1分支，如果不存在则报错
git switch -c branchName 创建branchName分支

gitk查看提交记录
