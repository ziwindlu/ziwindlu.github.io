Coc-nvim安装coc-java时遇到的一个问题

发表于2022-08-07|更新于2023-02-28|折腾日记
|字数总计:2.8k|阅读时长:15分钟|阅读量:

错误抛出

vim中报错为

排查

vim中报错很笼统，所以:CocInfo查看了以下coc的报错

具体报错忘记保存了，大概就是上面这个包jdk15之后不再使用，以后可能弃用

java --version显示现在用的java时jdk18,之前成功装过这个插件，那台机器时jdk17,于是决定将本机java切换到jdk17

解决

进入浏览器，how to change java version in linux，ENTER，得到的结果大多是通过update-alternatives命令切换，尝试后无果，猜测是arch弃用了相关命令，就像弃用ifconfig一样，但update-alternatives却存在于arch中。

换了个思路，how to change java version in arch linux -site:csdn.net，ENTER，不要在意为啥过滤了csdn，arch提供了一个脚本archlinux-java用于用户切换java版本。

看看咋用，archlinux-java --help，就简单的几个参数，status，set，get，unset，fix。

重新使用vim打开一个java文件，发现报错更多了。搜索后得知将该插件重新安装即可

进入vim，:CocInstall coc-java，一路顺通无阻，同志们记得挂代理

重新装coc里面的插件我不清楚用什么命令，就直接删除了

问题再现

重装完插件，当前文件是可以使用的，打开了另一个文件发现补全又炸了，:CocInfo

将该文本保存下来grep WARN error.txt发现了Suggest not triggered with input "", minimal trigger input length: 1搜索得知有小伙伴遇到了相同的问题

原来我们都装过coc-clang而且没有更新，尝试更新了插件就能用了:CocUpate

回顾

查报错，是linux用户的基本功，有机会学习学习

软件一定记得及时更新
